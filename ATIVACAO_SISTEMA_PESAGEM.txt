PORTAL SIMONSPORTS — SISTEMA DE PESAGEM NO GITHUB PAGES
Versão: 2026-07-13

ESTRUTURA IMPLEMENTADA
======================
1. index.html
   - Interface completa e responsiva.
   - Login por e-mail + código de acesso pelo Firebase Authentication.
   - Usuários pendentes, ativos e bloqueados.
   - Controle dos perfis permitidos.
   - Pesagem, desconto do celular, metas, água, histórico, gráficos e comparativos.

2. firebase/firestore.rules
   - O usuário comum cria somente a própria solicitação, inicialmente bloqueada.
   - O administrador portalsimonsports@gmail.com pode ativar usuários e definir perfis.
   - Nenhum dado de pesagem é armazenado no Firestore; o Firestore controla apenas acessos.

3. apps-script/PSS_PESAGEM_BRIDGE_COMPLETO.txt
   - Valida o token Firebase no servidor.
   - Valida o documento usuarios_pesagem/{uid}.
   - Restringe cada usuário aos perfis liberados.
   - Reutiliza as funções existentes do Sistema de Pesagem.

4. apps-script/bridge.html.txt
   - Ponte segura entre o GitHub Pages e google.script.run.
   - A tela principal permanece no GitHub; o Apps Script funciona somente como base/API.

ATIVAÇÃO NECESSÁRIA FORA DO GITHUB
==================================
A integração exige três configurações que não podem ser aplicadas por um commit no repositório:

A) FIREBASE AUTHENTICATION
- Firebase Console > Authentication > Sign-in method.
- Ativar E-mail/senha.

B) FIRESTORE
- Firebase Console > Firestore Database > Rules.
- Publicar o conteúdo de firebase/firestore.rules.

C) GOOGLE APPS SCRIPT
- No projeto atual do Sistema de Pesagem:
  1. Substituir a função doGet pela versão do arquivo PSS_PESAGEM_BRIDGE_COMPLETO.txt.
  2. Adicionar as demais funções desse arquivo.
  3. Criar o arquivo HTML "bridge".
  4. Colar nele o conteúdo de bridge.html.txt.
  5. Executar PSS_PESAGEM_DIAGNOSTICAR_BRIDGE.
  6. Implantar > Gerenciar implantações > Editar > Nova versão > Implantar.
- O endereço do Web App pode continuar o mesmo.

D) GITHUB PAGES
- Repositório > Settings > Pages.
- Source: GitHub Actions.

SEGURANÇA
=========
- O código de acesso é a senha do Firebase Authentication e não fica no HTML.
- O Apps Script valida o Firebase ID Token antes de ler ou alterar a planilha.
- A autorização por perfil também é aplicada no Apps Script, não apenas na interface.
- O sistema não expõe o segredo interno da planilha ao navegador.
